# 前言：What is Aelin

![[images/smile.png]]

## 为什么会创造Aelin?

2026年寒假，计算机应用能力大赛（也就是口口相传的计算机设计大赛）开始进入准备阶段了。
当时的我们正处在一个“刚获得能力与技术，却开始缺乏灵感”的空乏期。三个人的团队想了很久，在一些前辈的建议下，决心做一个叫做 `MercuryDesk`的项目。
![[images/MercuryDesk_logo.png]]

我们将 `MercuryDesk`设定为一个信息聚合平台，其中整合了各个平台的爬虫技术，能够主动获取多平台的更新，并整合在一起，呈现出一个杂志化面板：
![[images/MercuryDesk.png]]

随后出于我个人的技术栈，我们向其中接入了Agent功能，如消息总结、自动排序置顶、chat等等。在做chat时，我们做了极其精细的上下文工程，包括短期记忆、长期notes、跟踪帖子及其摆放位置（反应用户的重视程度）信息。渐渐的我们意识到chat agent的体量已经有点超乎 `MercuryDesk`本身，所以我们打算重新整合该项目，以chat agent为主要内容和入口。
——这就是Aelin的诞生。

## 为什么是黄毛妹

其实Aelin的黄毛形象是ChatGPT生成的，我们甚至还为她做了很多表情包~
当时让gpt成图的时候，想的是弄个萌一点的，漫画风的，但是弄半天都只生成了一些圆脸、没有啥感染力的纯情妹。可是咱们这个应该也算是个办公软件，熟女应该会合称一点（其实主要是作者本人喜欢熟女）。

于是就改了改提示词，加了个眼镜，还是不太OK，因为gpt成图出来的那个眼镜巨丑。
于是就干脆弄成黄毛，模拟《轻音少女》中的琴吹紬。然后就成这样了。但是它似乎完全没有继承琴吹的“浓眉”的精髓。。。但是这一点应该只有作者在乎，没有浓眉的黄毛妹应该更加符合大众的审美，所以就这样定了。
表情包一类的物件就是直接gpt生成的，无需多言。

## chat agent会不会没有意义？

其实我们的团队一直处在一种深度的“意义焦虑”之中，一直在为自己做的事情可能没有意义而感到恐慌。`MercuryDesk`，甚至之前还有一个草案叫做 `EditMason`，都由于“缺乏意义”而被我们废置了。
chat agent的意义分散在两个方面，一方面是协助办公，另一方面是情感支持。

Aelin本身是为办公而生的，她的爬取内容后能进行订阅，进行订阅后能进行本地化存储，以进入上下文的机制就反映了这一点。当然，她并没有像web上的chatGPT那样，可以开“代理模式”，可以接入各路软件，近乎无所不能。但她从架构和需求上与那些功能有明确划分，起码不会被覆盖掉。至于她会不会成为一个受欢迎的agent，一个受欢迎的软件，那就不可预知了~

Aelin有为了情感支持做出一些努力，最具有开创性的就是“表情包”的机制的存在，注意，不是emoji，而是说，Aelin有一套真实的属于自己的表情包。除此之外，Aelin的记忆机制、主动交互和 `answer first`的原则也为该点做出了努力（最近的openclaw和Alma似乎都有了做不到 `answer first`的问题存在）。

## Aelin的未来

Aelin的功能已经较为齐全了~，但是她的未来还会有更加强大的功能被推出，毕竟chat agent是一个很大的开发领域，而开发领域就是这样，有人做出了新的东西，你不做，那就完蛋了。

首先肯定要做“操纵电脑的能力”。但是在测试Aelin时我们使用的模型是 `seed 1.8`和 `deepseek chat`，这俩并不便于用来做开发“操纵电脑的能力”的载体，所以暂时没做。

然后肯定要做多端啦，咱们的团队目前只有windows设备，所以只能构建windows和安卓项目，之后等设备过关了，可以着手做mac和iphone上的包。

再之后，可能会做更优秀的主动交互，会做好token成本和体验优化之间的平衡，会尝试建立起更加合理的主动交互逻辑。

总之，这一切得建立在Aelin取得暂时性的认可、暂时性的成功的基础上。
