# 前言：What Is Aelin

Aelin 是一个面向个人场景的长期记忆型 AI 助手。它不是“问一次答一次”的单轮聊天工具，而是把问答、证据、追踪和记忆沉淀放在同一条产品链路里。

## 从 MercuryDesk 到 Aelin

当前实现来自 `MercuryDesk` 源码仓库，但产品命名和交互主轴已经切换到 Aelin：

- 产品层：以 Chat 为统一入口，面向长期决策与持续关注。
- 工程层：后端仍使用 `MERCURYDESK_*` 环境变量前缀，保持历史兼容。
- 交付层：支持 Web 与 Electron 桌面壳，强调本地部署可用性。

## 设计原则

1. Answer-first
先给可执行结论，再给证据与下一步动作。

2. Evidence-first
时间敏感问题优先给可追溯引用，避免“只给观点”。

3. Tracking-native
将“关注某对象”升级为可持续运行的追踪目标。

4. Memory-by-default
把高价值结果投影为可检索的本地 Markdown 和记忆层。

5. Local-first operations
在本地机上可独立运行，包括鉴权 fallback、设备控制和桌面壳。

## 核心架构分层

- 接入层：Accounts + Inbound，负责 OAuth、IMAP、RSS 与社媒连接器接入。
- 推理层：Aelin Chat，负责意图分析、规划、检索、校验与回答生成。
- 执行层：Tracking Autonomy，后台线程持续调度目标、比较快照并产出变化。
- 记忆层：数据库 + 文件记忆桥（OpenViking 可选），沉淀 profile/snapshot/change/diary。
- 交互层：React 前端 + Electron 桌面壳 + 桌宠引擎。

## 当前边界（源码可见）

- 前端保留知乎入口，但后端无独立知乎连接器。
- 设备模式只保证 `meeting/focus/sleep/normal/default`，其它值会归一到 `normal`。
- 无 token 时会回退本地单用户，适合个人本地部署，不等同于多租户 SaaS 鉴权模型。

带着这几个前提阅读后续文档，你会更容易判断 Aelin 能做什么、该如何落地、哪些场景需要额外兜底。
