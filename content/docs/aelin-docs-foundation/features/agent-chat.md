# Agent Chat

`Agent Chat` 是 Aelin 最常被直接感知的能力层。它看起来像聊天，但目标并不只是“对话自然”，而是把信息组织、结论表达与后续行动建议结合在同一条流程里。

## 核心体验

在日常使用中，你会感受到三件事：

- 回答节奏更偏“先结论后细节”，避免你先读很长铺垫。
- 证据会以可追溯的方式呈现，便于你判断可靠性。
- 对话是持续性的，不需要每次都把背景重新讲一遍。

这意味着它更像一个长期协作界面，而不是一次性问答窗口。

## Answer-first

Aelin 默认采用 `Answer-first`，即先给你可以立即使用的结果，再补充：

- 结论来自哪些证据；
- 还存在哪些不确定性；
- 下一步可以怎么做（例如继续追问或建立跟踪）。

这种结构的价值在于：你可以先做判断，再决定是否深入细节，而不是被迫从大量文字中手动提炼重点。

## 会话能力

Aelin 的会话能力强调“连续推进”：

- 多会话并行，适合不同主题分别管理；
- 会话历史本地持久化，减少中断损失；
- 可注入近期上下文与记忆摘要，让追问更精准。

当你第二次回到同一话题时，系统会尽量延续已有语境，减少重复沟通。

## 适合的使用场景

- 需要快速获取结论，但又希望随时查看依据。
- 问题会持续演化，需要多轮追问而非一次结束。
- 希望把“提问-判断-跟进”变成稳定流程。

## 使用建议

- 提问尽量具体，尤其是对象名称、时间范围和比较维度。
- 当系统给出跟踪建议时，结合你的真实长期需求再确认。
- 如果你需要更强可追溯性，优先关注证据卡片而不只看结论。

## 能力边界

`Agent Chat` 的表现仍受模型质量、数据可得性和平台限制影响。Aelin 会努力把这些不确定性显式呈现出来，但在关键信息场景下，仍建议你进行必要交叉核验。
