# Proactive Interaction

`Proactive Interaction` 的作用是把“你来问”变成“系统先提示”。

当你关注的对象发生重要变化时，Aelin 会在适当时机主动发出提醒，减少信息错过与重复检查的成本。

## 主动提醒来源

常见触发来源包括：

- 今日简报中的高价值更新。
- 跟踪任务状态变化（新增、异常、中断、恢复）。
- 未读堆积或信息密度异常。
- 设备状态异常（例如持续高占用）。

你不需要手动盯住每个入口，系统会先做一层筛选后再通知。

## 交互方式

- 顶部角标：快速感知“有新变化”。
- 通知中心：集中查看事件列表。
- 可执行动作：直接跳转到相关对话、跟踪或详情页。

这种设计的目的是减少路径跳转，让提醒不仅“看见”，还能“马上处理”。

## 防打扰策略

为了避免提醒过载，Aelin 默认启用以下机制：

- 去重：同类事件不重复轰炸。
- 节流：控制单位时间内提醒数量。
- 首次访问限流：避免初次进入即收到大量历史通知。

你会得到的是“更少但更有价值”的提醒，而不是频繁打断。

## 使用建议

- 对高优先级主题开启主动提醒，对低优先级主题保持被动查询。
- 定期回看通知中心，清理已完成事项，保持提醒质量。
- 如果某类提醒长期无价值，及时调整策略而不是硬抗噪声。

## 能力边界

主动提醒本质上是“概率最优策略”，并不保证覆盖所有变化。对于关键业务决策，建议仍结合固定节奏的主动检查，以确保信息完整性。
