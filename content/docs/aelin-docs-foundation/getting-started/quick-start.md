# Quick Start

这一节的目标很简单：不讲太多概念，先把 Aelin 用起来。

你只需要按下面的步骤走一遍，就能直观看到它和普通聊天工具的差异。

## 1. 启动服务并确认账号

- 启动后端与前端服务。
- 进入页面并登录。
- 确认 `Agent Config` 中已经配置可用模型（例如 DeepSeek 或其他兼容提供商）。

建议在这里多做一步：先执行一次连通性测试。这样后面遇到异常时，你可以更快判断是“模型链路问题”还是“业务逻辑问题”。

## 2. 发送第一条问题

你可以从一个“有时效性 + 有后续价值”的问题开始，例如：

```text
今天勇士和马刺的比赛结果如何？
```

在这个过程中，Aelin 会自动决策使用哪种检索路径：

- 本地检索（你已有历史）
- 网络检索（外部最新信息）
- 两者并行融合（更完整）

## 3. 观察回答结构

你会看到回答通常分为三层：

- 先给出可用结论（Answer-first）
- 再补证据来源
- 最后给下一步建议（例如是否建立跟踪）

这一步非常关键：Aelin 的设计目标不是“把话说得很多”，而是“先把决策关键信息给到你”。

## 4. 确认是否建立跟踪

当系统判断你对某个对象有持续关注价值时，会给出跟踪建议：

- 选择“跟踪”：进入长期任务，后续变化会被持续抓取与更新。
- 选择“仅这次”：只完成当前问题，不进入持续流程。

你始终拥有最终确认权，Aelin 不会绕过你的决策自动创建长期任务。

## 5. 查看沉淀结果

你可以在以下位置看到系统沉淀：

- 对话中的证据卡片
- 跟踪中心状态
- 记忆层（`facts / preferences / in_progress`）

这一步能帮助你理解：Aelin 的价值不只在“答一次”，而在“让下一次更轻松”。

## 6. 发起第二次追问

例如：

```text
那库里最近 5 场状态有没有波动？
```

这时，系统会在已有上下文之上继续推进，而不是把你当作第一次来访者。

## 7. 一次闭环完成后你可以做什么

完成以上步骤后，建议你继续看：

- `Features`：理解每个能力模块的工作边界
- `Guides`：把常见流程用更稳定的方式跑通
- `Reference`：遇到问题时快速定位接口与机制

如果你愿意，下一次可以换一个你真正长期关心的主题（人物、项目、行业动态）再做一轮，你会更明显地感受到“长期助手”的差异。
