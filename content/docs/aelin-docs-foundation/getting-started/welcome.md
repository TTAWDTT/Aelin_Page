# Welcome to Aelin

Aelin 的目标不是替代搜索框，而是把“持续关注 + 持续判断”这件事做成可运行的系统。

## Aelin 在解决什么问题

现实里的信息管理难点通常不是“找不到一条消息”，而是：

- 同一主题分散在多个平台，来回切换成本高。
- 普通 Chat 只记住短期对话，长期价值难沉淀。
- 你知道该关注什么，但缺少稳定的自动跟踪机制。

Aelin 的做法是把一次提问延长成一条链路：

`提问 -> 证据 -> 追踪 -> 变化 -> 通知 -> 日记记忆`

## 你可以直接使用的核心能力

- Chat 主入口：SSE 流式返回、引用来源、动作建议、链路可视化。
- 多源接入：邮件、OAuth 平台、RSS 与社媒源统一接入与同步。
- 长期追踪：支持自动轮询与手动触发，变化可确认已读。
- Desk 观察台：跨来源内容流，支持标签和关键词检索。
- 文件记忆与日记：自动落地到本地 Markdown，可树状浏览与全文读取。
- 本地设备联动：进程治理、设备模式、专注静默。

## Aelin 与普通 AI Chat 的区别

- 普通 Chat 偏“即时问答”。
- Aelin 偏“持续协作”，关注后续变化和可复用记忆。

所以你第二次回来时，不需要从零解释背景，系统会基于既有上下文继续推进。
