# Create a Tracking Flow

如果你想真正发挥 Aelin 的长期价值，这篇指南非常关键。

我们会把一个普通提问，逐步转成可持续跟踪任务，并确保你能看见每一步状态。

## 示例流程

1. 你发起问题：`最近某个关注对象有什么新动态？`
2. Aelin 给出结论与证据。
3. 系统询问是否建立长期跟踪。
4. 你确认后，系统检查数据源配置。
5. 跟踪开始，后续变化持续写入相关模块。

这个流程看起来简单，但核心价值在于：你不再需要重复追问同一主题。

## 如何让跟踪更稳定

- 目标尽量具体：账号、链接、唯一标识优先。
- 范围不要过大：先小范围验证，再逐步扩展。
- 主题分层管理：不同项目用不同 workspace。

越清晰的输入，越稳定的后续更新。

## 状态观察点

跟踪建立后，建议你重点看三个位置：

- 跟踪中心：任务是否正常运行。
- 通知中心：有没有关键变化。
- 记忆层：信息是否被正确沉淀。

如果其中某一层长期没有更新，通常意味着你需要检查数据源或调整目标定义。

## 常见误区

- 目标过于宽泛，导致噪声过高。
- 关注对象变更后未更新配置。
- 建立了大量低价值跟踪，稀释有效提醒。

你可以把跟踪任务当作“资产”，而不是“越多越好”的清单。

## 推荐节奏

- 每周快速检查一次任务健康度。
- 每月清理一次低价值或失效目标。
- 对关键主题保留长期轨迹，形成连续判断能力。

这样你会逐步从“查信息”过渡到“管理信息流”。
