# Known Issues

## 1. 知乎连接器未落地

现状：前端设置页保留知乎入口，但后端没有独立知乎连接器。

影响：直接按“知乎”接入无法形成完整同步链路。

建议：优先使用 RSS/RSSHub 订阅路径。

## 2. 设备模式存在明确支持边界

现状：后端只保证 `meeting/focus/sleep/normal/default`。

影响：其它模式值会被归一，表现为降级处理。

建议：业务侧模式枚举与后端保持一致。

## 3. 非 Windows 环境设备控制会降级

现状：系统级动作（通知、亮度、进程优先级）以 Windows 为主实现。

影响：在其它平台可能仅返回状态，不保证系统级执行。

建议：跨平台场景加入人工兜底流程。

## 4. 鉴权模型偏本地化

现状：无 token 时会回退本地单用户。

影响：默认行为更适合本地个人部署，不适合直接当公网多租户鉴权方案。

建议：对外部署时补充网关鉴权与账号体系隔离。

## 5. 社媒抓取稳定性受上游限制

现状：部分平台依赖公开页面/接口，受反爬、结构变更、登录态影响。

影响：可能出现抓取失败、数据延迟、结果波动。

建议：

- 关键目标开启多源交叉
- 按平台配置 cookies/OAuth
- 对关键任务保留手动 run 校验

## 6. 媒体摄取依赖本机工具链

现状：需要 `yt-dlp`（及部分场景下 `ffmpeg`）。

影响：缺少依赖时会直接返回 tool_missing/extract_failed。

建议：在部署自检中把媒体工具链纳入必检项。

## 7. 桌面打包依赖环境一致性

现状：Electron 打包受 Node/Python/证书/代理环境影响。

影响：首次打包可能失败或产物不可运行。

建议：固定版本、保留验证脚本并建立可回滚构建流程。
