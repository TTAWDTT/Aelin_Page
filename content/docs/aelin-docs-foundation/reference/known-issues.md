# Known Issues

这部分会持续更新。你可以把它当作“当前版本边界说明”，帮助你在实践中更早规避问题。

## 1. 外部网站抽取稳定性波动

部分站点存在反爬、动态渲染或结构频繁变更，会导致正文抽取质量波动。

建议：

- 对关键来源做多源交叉。
- 对结构不稳定站点降低依赖权重。

## 2. 设备控制存在平台差异

设备侧动作在 Windows 上通常更完整，其他系统可能只支持部分能力或仅返回状态信息。

建议：

- 在目标平台做能力预检。
- 为关键动作保留人工兜底流程。

## 3. 移动端网络配置敏感

安卓模拟器或真机必须能访问后端地址，否则会出现 `failed to fetch`。

建议：

- 确认同网段、端口开放和防火墙策略。
- 使用可直连地址替代本地回环地址。

## 4. 桌面打包依赖环境

首次打包依赖较多，网络代理、证书策略或 SDK 版本不一致都可能影响流程。

建议：

- 先完成环境自检再打包。
- 固定依赖版本并保留可回滚方案。

## 5. 首次开发态访问可能偏慢

在开发模式下，首次进入某些页面可能触发编译与预热开销。

建议：

- 使用预热机制减少首跳等待。
- 对真实性能评估优先使用生产构建环境。

我们会持续收敛上述问题，并在版本更新中同步进展。
