# Release Notes 2026 Q1

这是 Aelin 文档体系的首个里程碑版本。我们把目标放在一件事上：让用户更平稳地理解产品能力，并能在真实场景中持续用起来。

## 文档结构升级

本次更新完成了从“零散说明”到“产品化文档结构”的迁移：

- 建立统一信息架构。
- 拆分为 Getting Started / Concepts / Features / Guides / Reference / Release Notes。
- 统一路由命名与阅读顺序。

这使得新用户可以更快上手，老用户也能更高效回查。

## 核心价值表达统一

我们在文档叙事上统一了三条主线：

- `Answer-first`
- `Tracking`
- `Memory`

这三条主线对应 Aelin 的核心产品逻辑：先解决当下问题，再支持持续推进，最终沉淀为可复用上下文。

## 体验层改进

- 文档页导航与内容组织更清晰。
- 代码块与内容样式更统一。
- 首次加载与访问路径进行了性能优化。

这些改动并不追求“视觉花哨”，而是强调长时间阅读和持续使用时的稳定感。

## 后续计划

下一阶段会继续完善：

- 增加更多真实场景示例（不仅是功能列表）。
- 增加接口请求/响应示例与排障模板。
- 强化版本变更记录的自动化流程。

如果你在使用中发现文档盲区，欢迎直接反馈。我们会把“可执行、可回查、可持续”作为后续更新的核心标准。
